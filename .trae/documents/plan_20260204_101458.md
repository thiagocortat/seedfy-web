# Implementar Seedfy Web App (PRD-WEB-01)

Este plano descreve a transformação da atual `apps/landing` no aplicativo principal `apps/web`, implementando a estrutura de autenticação e área logada conforme o PRD.

## 1. Estrutura e Refatoração
### Transformar `apps/landing` em `apps/web`
1.  **Renomear diretório**: Mover `apps/landing` para `apps/web`.
2.  **Atualizar Identidade**: Ajustar `package.json` para nome `web` e porta padrão (ex: 3000 ou 3002).
3.  **Reorganizar Rotas (Route Groups)**:
    *   Mover páginas atuais (Home, Terms, Privacy) para `src/app/(landing)/`.
    *   Criar grupo `src/app/(auth)/` para rotas de autenticação.
    *   Criar grupo `src/app/(app)/` para rotas protegidas.

## 2. Autenticação (Supabase)
### Implementação do Fluxo de Auth
1.  **Páginas**:
    *   `/login`: Formulário de email/senha.
    *   `/signup`: Criação de conta.
    *   `/forgot-password`: Recuperação de senha.
2.  **Integração**: Utilizar `@seedfy/shared` (`supabase-browser`) para chamadas de auth no client.
3.  **Middleware**:
    *   Criar `middleware.ts` para proteção de rotas.
    *   Redirecionar `/app/*` para `/login` se não autenticado.
    *   Redirecionar `/login` para `/app` se já autenticado.

## 3. Área Logada (Shell)
### Estrutura do Dashboard (`/app`)
1.  **Layout (`(app)/layout.tsx`)**:
    *   **Sidebar**: Navegação principal (Home, Conteúdos, Grupos, Desafios, Perfil).
    *   **Topbar**: Ações de usuário e menu mobile.
    *   **Responsividade**: Drawer/Menu para mobile.
2.  **Páginas Placeholder**:
    *   `/app/page.tsx` (Dashboard/Home).
    *   `/app/content/page.tsx`, `/app/groups/page.tsx`, etc. (Empty states).

## 4. Dependências
*   Reutilizar `@seedfy/ui` e `@seedfy/shared`.
*   Manter consistência com `apps/admin` (Next.js 16, React 19).

Este plano garante a entrega dos "Fundamentos" sem alterações no banco de dados, focando na experiência do usuário e arquitetura segura.
